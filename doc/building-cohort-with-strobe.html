Building Cohorts with STROBE Diagrams

Introduction

The STROBE (Strengthening the Reporting of Observational Studies in Epidemiology) statement provides guidelines for reporting observational studies. A key component is the flow diagram that shows how the study population was selected. The sRtr package provides tools to create STROBE-compliant flow diagrams while building your cohort.

This vignette demonstrates how to use the STROBE functions in sRtr to:

  1. Initialize a cohort with inclusion criteria
  2. Apply sequential filters with exclusion tracking
  3. Generate and customize STROBE flow diagrams
  4. Review the complete filtering log

Getting Started

library(sRtr)
library(dplyr)

Basic STROBE Workflow

The STROBE workflow in sRtr follows these steps:

  1. Initialize the cohort with strobe_initialize()
  2. Filter sequentially with strobe_filter()
  3. Plot the diagram with plot_strobe_diagram()
  4. Review the log with get_strobe_log()

Reviewing the Filtering Log

# View the complete filtering log
get_strobe_log()

The log shows: - Each filtering step - Number of records included/excluded at each step - Cumulative counts - Exclusion reasons

Creating STROBE Diagrams

Basic Diagram

plot_strobe_diagram()

Customizing Box Dimensions

plot_strobe_diagram(
  incl_width_min = 3,
  incl_height = 1,
  excl_width_min = 2.5,
  excl_height = 1,
  lock_width_min = TRUE,
  lock_height = TRUE
)

plot_strobe_diagram(
  incl_width_min = 3,
  excl_width_min = 2.5,
  incl_fontsize = 16,
  excl_fontsize = 14,
  lock_width_min = TRUE
)

Summary

The STROBE functions in sRtr provide a systematic way to:

  • Document your cohort selection process
  • Create publication-ready flow diagrams
  • Ensure transparent reporting of observational studies
  • Maintain a complete audit trail of filtering decisions

Key features:

  • Flexible box sizing
  • Direct export to PNG/SVG
  • Customizable styling
  • STROBE-compliant layout
